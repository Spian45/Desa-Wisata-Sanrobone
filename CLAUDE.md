# CLAUDE.md

Guidance for working in this repository.

## What this is

A **static, multi-page marketing website** for *Desa Wisata Sanrobone* — a
cultural/heritage tourism village in Kabupaten Takalar, South Sulawesi,
Indonesia. Plain HTML + CSS + a little jQuery. No build step, no framework, no
package manager. Open an `.html` file in a browser to run it.

All UI text is in **Indonesian (`lang="id"`)**.

## Layout

| File | Role |
|------|------|
| `index.html` | Main landing page (hero video, sejarah, potensi, akomodasi, amenitas, souvenir, galeri, reservasi). Has its own inline `<style>` block on top of `style.css`. |
| `detail-potensi.html` | Detail page for tourism potential. |
| `akomodasi-detail.html` | Homestay / accommodation detail. |
| `amenitas-detail.html` | Facilities detail. |
| `souvenir-detail.html` | Souvenir / oleh-oleh detail. |
| `info-desa.html` | Village information page. |
| `style.css` | Shared global stylesheet (layout, nav, sections, footer, responsive). |
| `javascript.js` | jQuery: mobile hamburger menu toggle + navbar color change on scroll. |
| `asset/` | Images, the hero video (`video-indonesia.mp4`), icons. |

## Conventions

- **Indonesian class names** are intentional: `layar-dalam` (inner container,
  fixed `1000px` wide), `layar-penuh` (full width), `tombol` (button),
  `tombol-menu` (hamburger), `abuabu` (grey-background section), `ringkasan`
  (subtitle), `konten-isi` (body content), `tim` (card grid), `support`
  (icon-card grid).
- **Brand accent color:** `#c86b85` (used for buttons, underlines, highlights).
- **Section pattern:** each `<section id="...">` alternates plain / `.abuabu`
  grey background; headings get an auto-centered accent underline via
  `section h3::after`.
- The fixed navbar is transparent at the top and gains the `.putih` (white)
  class once scrolled — toggled in `javascript.js`.
- `index.html` keeps page-specific tweaks in its inline `<style>` block rather
  than editing `style.css`, so changes there don't affect the other pages.

## Editing guidance

- **Preserve the Indonesian class names and copy** — don't anglicize them.
- Prefer editing the **inline `<style>` in `index.html`** for landing-page-only
  visual changes; edit `style.css` only for changes that should apply to every
  page.
- Always confirm referenced assets exist in `asset/` before adding `<img>`/
  `src` — some placeholder assets have been deleted from the repo.
- Keep the responsive breakpoints consistent: `style.css` uses `max-width:
  991.98px` for the mobile nav; `index.html` adds a `max-width: 768px` block.
- No tooling/tests — verify changes by opening the page in a browser.

## Known constraints / gotchas

- `javascript.js` scroll handler also targets `nav img.hitam` / `nav img.putih`,
  but `index.html` uses **text** logos (`.logo .putih` / `.logo .hitam`), whose
  show/hide is handled purely by CSS. The image selectors are harmless no-ops.
- The QR code in the Reservasi section is generated by an **external API**
  (`api.qrserver.com`) and requires internet access to render.
- `.layar-dalam` is a hard `1000px`; it only becomes fluid (`90%`) below the
  `991.98px` breakpoint.
